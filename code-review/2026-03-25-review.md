# Code Review - YD YouTube Downloader

**Date:** 2026-03-25
**Scope:** Security, UI/UX, Accessibility
**Status:** Fixed

---

## Security Issues

### Critical

| # | Issue | Location | Status |
|---|-------|----------|--------|
| 1 | CSP disabled (`null`) - vulnerable to XSS | `tauri.conf.json` | Fixed |
| 2 | No checksum verification for yt-dlp in CI build | `release.yml` | Fixed |
| 3 | HTTP URLs accepted (insecure transport) | `download.rs`, `UrlInput.tsx` | Fixed |

### High

| # | Issue | Location | Status |
|---|-------|----------|--------|
| 4 | Unsafe FFI `i32` cast in `decode_output` - integer truncation risk | `download.rs` | Fixed |
| 5 | `taskkill` uses `.spawn()` (fire-and-forget) - no error checking | `download.rs` | Fixed |

### Medium

| # | Issue | Location | Status |
|---|-------|----------|--------|
| 6 | Overly broad `$HOME/**` file path permission | `capabilities/default.json` | Fixed |
| 7 | Temp file leak in `update_ytdlp` on error | `download.rs` | Fixed |
| 8 | URL length not bounded (potential abuse) | `download.rs` | Fixed |

### Not Fixed (Low Risk / Needs Discussion)

| # | Issue | Reason |
|---|-------|--------|
| 9 | ffmpeg metadata title not sanitized | ffmpeg treats `-metadata` value as literal; injection not exploitable since it's passed as a single arg, not shell-expanded |
| 10 | `.expect()` calls for app paths | These paths are guaranteed by Tauri runtime; converting to `Result` would add complexity for an impossible failure case |

## Code Quality Notes

### Positive

- Strong component separation with clear responsibilities
- Good memoization and event cleanup patterns
- Proper TypeScript strict mode
- Good i18n system with full Hebrew/English support
- RTL support works correctly
- Semaphore-based download rate limiting
- SHA256 verification for yt-dlp runtime updates
- rustls-tls instead of OpenSSL
- Input validation allowlists in `config.rs`

### Recommendations for Future

1. Add error boundaries for component crash recovery
2. Consider structured logging instead of `console.error`/`eprintln!`
3. Document lock ordering for `DownloadState` mutexes
4. Add `SIGKILL` fallback on Unix if `SIGTERM` doesn't terminate within timeout
